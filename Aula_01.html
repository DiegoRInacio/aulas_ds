💡 Big Data ≠ volume apenas

Big Data é sobre a incapacidade das ferramentas tradicionais de processar, armazenar ou analisar os dados — independentemente do tamanho absoluto.

Os 5 V's do Big Data

Framework
V
Volume
Quantidade massiva de dados gerados (TBs, PBs, ZBs)
V
Velocidade
Dados gerados e processados em alta velocidade ou tempo real
V
Variedade
Diferentes formatos: textos, imagens, vídeos, sensores, logs
V
Veracidade
Qualidade, confiabilidade e consistência dos dados coletados
V
Valor
Capacidade de transformar dados em insights e decisões de negócio

🎯 Lembre-se

Os 5 V's são um diagnóstico, não uma meta. Entender em qual V está o desafio da organização direciona a solução tecnológica e analítica mais adequada.

🔄 É um ciclo, não uma linha

Os dados alimentam decisões que geram novos dados. A qualidade em cada etapa impacta todas as seguintes — lixo entra, lixo sai (Garbage In, Garbage Out).

⚠️ Gargalos comuns

  • Dados coletados mas nunca utilizados
  • Armazenamento sem organização ou metadados
  • Análise sem contexto de negócio
  • Insights sem canal de decisão

Tipos de Dados

Taxonomia

📋 Estruturados

Organizados em tabelas com linhas e colunas. Fácil para consulta e análise.

  • Bancos relacionais (SQL)
  • Planilhas (Excel, CSV)
  • Sistemas ERP/CRM
~20% dos dados corporativos

📄 Semiestruturados

Possuem alguma organização, mas não seguem esquema tabular rígido.

  • JSON / XML
  • Logs de sistemas
  • E-mails com metadados
  • HTML
~15% dos dados corporativos

🖼️ Não Estruturados

Sem esquema predefinido. Exigem técnicas avançadas para extração de valor.

  • Imagens e vídeos
  • Textos livres / PDFs
  • Áudios e podcasts
  • Redes sociais
~65% dos dados corporativos

🚀 O desafio moderno

A maior parte do valor latente está nos dados não estruturados — exatamente os mais difíceis de processar. LLMs e visão computacional estão mudando isso rapidamente.

Qualidade & Governança de Dados

Gestão

✅ Dimensões da Qualidade

  • Completude: Campos obrigatórios preenchidos?
  • Consistência: Mesmo dado em fontes diferentes bate?
  • Precisão: O dado representa a realidade?
  • Atualidade: O dado está desatualizado?
  • Unicidade: Há duplicatas?
  • Validade: Está no formato e range esperado?

🏛️ Pilares da Governança

  • Catalogação: Metadados e dicionário de dados
  • Linhagem: De onde veio o dado e como foi transformado
  • Acesso: Quem pode ver e modificar quais dados
  • Privacidade: LGPD/GDPR, anonimização, consentimento
  • Responsabilidade: Data owners e stewards

⚖️ LGPD no contexto de negócios

A Lei Geral de Proteção de Dados (Brasil, 2020) impõe obrigações sobre coleta, armazenamento e uso de dados pessoais. Organizações que tratam Big Data precisam incorporar privacidade desde o design (Privacy by Design).

Big Data em ação — Casos reais

Cases

🛒 Amazon — Recomendação Preditiva

35% da receita vem do motor de recomendação, que processa trilhões de eventos de navegação para sugerir produtos em tempo real.

80% do conteúdo assistido vem de recomendações. Dados de comportamento de 260M+ assinantes guiam até decisões de produção de séries originais.
